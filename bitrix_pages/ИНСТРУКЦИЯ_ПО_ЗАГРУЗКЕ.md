#### 1.1. Создать папку `/about`
1. Находясь в корне сайта `/`, нажмите **"Создать раздел"**
2. Введите название: `about`
3. Нажмите **"Сохранить"**

#### 1.2. Создать папку `/contacts`
1. Находясь в корне сайта `/`, нажмите **"Создать раздел"**

#### 1.4. Создать папку `/guarantee`

## ⚠️ Важные замечания

### 1. Кодировка файлов
Убедитесь, что все файлы сохранены в кодировке **UTF-8 без BOM**. Иначе могут быть проблемы с отображением русских символов.

### 2. Права доступа
Файлы должны иметь права:
- Файлы `.php`: **644**
- Папки: **755**

### 3. Очистка кеша
После загрузки всех страниц обязательно очистите кеш:
1. Перейдите: **Настройки → Производительность → Управление кешем**
2. Нажмите **"Очистить весь кеш"**

### 4. Проверка ссылок в меню
Убедитесь, что ссылки в `header.php` и `footer.php` указывают на правильные страницы:

```php
<a href="/about/">О компании</a>
<a href="/contacts/">Контакты</a>
<a href="/delivery/">Доставка</a>
<a href="/guarantee/">Гарантия</a>
<a href="/privacy/">Политика конфиденциальности</a>
```

## 🆘 Возможные проблемы и решения

### Проблема 1: Страница не открывается (404)
**Решение:**
- Проверьте, что файл называется именно `index.php` (не `about_index.php`)
- Проверьте, что папка создана в корне сайта
- Очистите кеш Bitrix

### Проблема 2: Не применяются стили
**Решение:**
- Проверьте, что CSS файлы загружены в `/local/templates/dveryaninov/css/`
- Проверьте код в `header.php` — должны быть строки с `Asset::getInstance()->addCss()`
- Очистите кеш браузера (Ctrl+F5)

### Проблема 3: Не отображаются изображения
**Решение:**
- Проверьте, что изображения загружены в `/local/templates/dveryaninov/images/`
- Проверьте пути в коде — они должны использовать `<?= SITE_TEMPLATE_PATH ?>/images/`
- Проверьте права доступа к файлам

### Проблема 4: Кракозябры вместо русских букв
**Решение:**
- Пересохраните файл в кодировке UTF-8 без BOM
- Проверьте мета-тег `<meta charset="UTF-8">` в header.php
