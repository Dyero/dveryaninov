### 🎯 Основное изменение
Изменён подход к работе с изображениями: вместо хранения путей в виде строк теперь используются поля типа **"Файл"**.

### ✅ Преимущества нового подхода

1. **Удобство загрузки**
   - Изображения загружаются прямо в админ-панели при редактировании записи
   - Не нужно предварительно загружать файлы в `/upload/dveryaninov/`
   - Битрикс сам управляет хранением файлов

2. **Простота добавления новых позиций**
   - Добавили новое покрытие? Просто загрузите картинку через форму редактирования
   - Не нужно возиться с файловым менеджером и путями

3. **Надёжность**
   - Битрикс сам следит за корректностью ссылок
   - Автоматическое создание превью и оптимизация изображений

4. Выбрать изображение с локального диска
5. Сохранить

### ❓ Часто задаваемые вопросы

**Q: Нужно ли удалять старые записи с путями?**
A: Если вы уже импортировали CSV со старыми путями, лучше удалить эти записи и импортировать заново без путей.

**Q: Можно ли использовать API для массовой загрузки изображений?**
A: Да, можно написать PHP-скрипт, который будет загружать файлы через API Битрикса и привязывать их к записям. Но это потребует навыков программирования.

**Q: Какой формат изображений поддерживается?**
A: Битрикс поддерживает JPG, PNG, GIF, WebP. Рекомендуется использовать JPG для фотографий и PNG для графики.

**Q: Какой размер изображений оптимален?**
A: Для покрытий и стёкол: 800x800px. Для дверей: 1200x1600px. Битрикс автоматически создаст превью нужных размеров.
