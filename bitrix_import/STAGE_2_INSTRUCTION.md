## ЧАСТЬ 1: ИМПОРТ HIGHLOAD БЛОКОВ

### 1.1 Импорт покрытий (96 записей)

**Шаг 1:** Откройте Highload блок "Покрытия"
- Путь: Админка → Настройки → Настройки продукта → Настройки модулей → Highload-блоки
- Выберите блок "Покрытия" (или создайте, если ещё не создан)

**Шаг 2:** Настройте поля блока (если ещё не настроены):
```
Поле                 | Тип               | Обязательное | Уникальное
---------------------|-------------------|--------------|------------
UF_XML_ID           | Строка            | Да           | Да
UF_NAME             | Строка            | Да           | Нет
UF_COLOR_HEX        | Строка            | Нет          | Нет
UF_TYPE             | Список (pvc,pet,enamel,custom) | Да | Нет
UF_IMAGE            | Файл              | Нет          | Нет
UF_SORT             | Число             | Нет          | Нет
UF_ACTIVE           | Да/Нет            | Да           | Нет
```

**Шаг 3:** Импорт CSV
1. Нажмите "Импорт данных"
2. Загрузите файл `1_coatings_import.csv`
3. Настройте соответствие столбцов:
   - CSV: `UF_XML_ID` → Поле: `UF_XML_ID`
   - CSV: `UF_NAME` → Поле: `UF_NAME`
   - CSV: `UF_COLOR_HEX` → Поле: `UF_COLOR_HEX`
   - CSV: `UF_TYPE` → Поле: `UF_TYPE`
   - CSV: `UF_SORT` → Поле: `UF_SORT`
   - CSV: `UF_ACTIVE` → Поле: `UF_ACTIVE`
4. Запустите импорт
5. Проверьте: должно быть импортировано 96 записей

**Шаг 4:** Привязка изображений
После импорта нужно вручную привязать изображения к записям:
1. Откройте список элементов Highload блока "Покрытия"
2. Для каждой записи:
   - Откройте элемент на редактирование
   - В поле `UF_IMAGE` нажмите "Загрузить файл"
   - Выберите соответствующее изображение из папки `images/Покрытия/ПВХ/`, `images/Покрытия/ПЭТ/` или `images/Покрытия/Эмаль/`
   - Сохраните запись

**СОВЕТ:** Начните с популярных покрытий (Белый матовый, Белая эмаль, и т.д.) для проверки работоспособности конфигуратора

## ВАЖНЫЕ ЗАМЕЧАНИЯ

1. **Кодировка CSV:** Все CSV-файлы в кодировке UTF-8. При открытии в Excel может потребоваться импорт с указанием кодировки.

2. **Разделители:** В CSV используется точка с запятой (;) как разделитель полей.

3. **Изображения:** Пути к изображениям в CSV указаны относительно корня сайта. После загрузки изображений в Битрикс нужно обновить ссылки или использовать скрипт привязки.

4. **Множественные значения:** Поле COATING_TYPES содержит несколько значений через запятую (например: "pvc,pet,enamel"). Битрикс может потребовать создания множественного свойства или разбора строки в JS.

5. **Цены:** Базовые цены указаны в рублях. Цена ПО = Цена ПГ + 14200 ₽.

6. **Тип полотна BOTH:** Означает, что дверь доступна и в ПГ, и в ПО вариантах. В конфигураторе пользователь сможет выбрать.

## ЧТО ДЕЛАТЬ ЕСЛИ ЧТО-ТО ПОШЛО НЕ ТАК

### Проблема: Импорт не запускается
**Решение:** Проверьте формат CSV, кодировку UTF-8, разделитель - точка с запятой

### Проблема: Не все записи импортировались
**Решение:** Проверьте лог импорта, возможно есть дубликаты по XML_ID или нарушены обязательные поля

### Проблема: Изображения не привязываются
**Решение:**
1. Убедитесь что изображения загружены в /upload/dveryaninov/
2. Проверьте пути в CSV - они должны совпадать с реальными путями
3. Используйте скрипт автоматической привязки

### Проблема: Двери импортировались не в те разделы
**Решение:**
1. Проверьте что разделы созданы с правильными символьными кодами
2. Убедитесь что поле COLLECTION в CSV совпадает с кодом раздела
3. Перезапустите импорт с правильным соответствием полей
