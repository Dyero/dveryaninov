# Шаг 1: Настройка шаблона Bitrix

## Цель этапа

Создать и активировать шаблон сайта в Bitrix, скопировать все стили, скрипты и изображения.

---

## 1.1 Создание структуры шаблона

### На вашем сервере Bitrix создайте папки:

```
/home/bitrix/www/local/templates/dveryaninov/
├── header.php          ← Системный файл (открывает <html>, <body>)
├── footer.php          ← Системный файл (закрывает </body>, </html>)
├── includes/
│   ├── header.php      ← HTML шапки сайта
│   └── footer.php      ← HTML футера сайта
├── css/                ← Все CSS файлы
├── js/                 ← Все JavaScript файлы
├── images/             ← Все изображения
└── description.php     ← Описание шаблона (необязательно)
```

---

## 1.2 Копирование файлов на сервер

### Файлы из репозитория → на сервер:

| Из репозитория | Куда на сервере |
|----------------|-----------------|
| `css/` (вся папка) | `/local/templates/dveryaninov/css/` |
| `js/` (вся папка) | `/local/templates/dveryaninov/js/` |
| `images/` (вся папка) | `/local/templates/dveryaninov/images/` |

### Способы копирования:

**Вариант A: FTP/SFTP**
- Используйте FileZilla или другой FTP-клиент
- Подключитесь к серверу
- Загрузите папки `css`, `js`, `images`

**Вариант B: SSH**
```bash
# Подключитесь к серверу по SSH
ssh user@your-server.com

# Перейдите в папку шаблона
cd /home/bitrix/www/local/templates/dveryaninov/

# Загрузите файлы (используйте scp или rsync)
```

---

## 1.3 Создание файлов шаблона

### 📄 Файл: `/local/templates/dveryaninov/header.php`

Скопируйте содержимое из репозитория:
- Файл в репозитории: `/local/templates/dveryaninov/header.php`

**Проверьте, что файл содержит:**
- `<!DOCTYPE html>` и открытие `<html>`, `<head>`, `<body>`
- `$APPLICATION->ShowHead()`
- Подключение CSS и JS через `Asset::getInstance()->addCss()`
- `$APPLICATION->ShowPanel()` для админской панели
- **НЕ содержит** включения `includes/header.php` (это будет на страницах)

---

### 📄 Файл: `/local/templates/dveryaninov/footer.php`

Скопируйте содержимое из репозитория:
- Файл в репозитории: `/local/templates/dveryaninov/footer.php`

**Проверьте, что файл содержит:**
- Модальное окно авторизации
- Закрывающие теги `</body>` и `</html>`
- **НЕ содержит** включения `includes/footer.php` (это будет на страницах)

---

### 📄 Файл: `/local/templates/dveryaninov/includes/header.php`

Скопируйте содержимое из репозитория:
- Файл в репозитории: `/local/templates/dveryaninov/includes/header.php`

**Что там должно быть:**
- HTML-разметка шапки сайта (`<header>`)
- Навигация, логотип, иконки
- Использование `SITE_TEMPLATE_PATH` для путей к изображениям

---

### 📄 Файл: `/local/templates/dveryaninov/includes/footer.php`

Скопируйте содержимое из репозитория:
- Файл в репозитории: `/local/templates/dveryaninov/includes/footer.php`

**Что там должно быть:**
- HTML-разметка футера (`<footer>`)
- Контакты, ссылки, копирайт

---

## 1.4 Активация шаблона в Bitrix

1. Войдите в админ-панель Bitrix
2. Перейдите: **Настройки → Настройки продукта → Настройки модулей**
3. Выберите **Главный модуль**
4. Вкладка **"Сайты"**
5. Нажмите на ваш сайт (обычно `s1`)
6. В поле **"Шаблон по умолчанию"** выберите `dveryaninov`
7. Нажмите **"Сохранить"**

---

## 1.5 Проверка

### Откройте главную страницу сайта в браузере

**✅ Должно быть:**
- Админская панель Bitrix вверху страницы (если вы авторизованы)
- Белая страница или базовый HTML без контента (это нормально)
- В исходном коде страницы (Ctrl+U) видны подключения CSS и JS

**❌ Не должно быть:**
- Ошибок 404 на CSS/JS файлы
- Ошибок PHP
- Дублирования шапки или футера

---

## 1.6 Устранение проблем

### Проблема: CSS/JS не загружаются (ошибка 404)

**Решение:**
```bash
# Проверьте права доступа к папкам
chmod -R 755 /home/bitrix/www/local/templates/dveryaninov/css/
chmod -R 755 /home/bitrix/www/local/templates/dveryaninov/js/
chmod -R 755 /home/bitrix/www/local/templates/dveryaninov/images/
```

### Проблема: Ошибка "Fatal error: memory exhausted"

**Решение:**
- Убедитесь, что в `header.php` и `footer.php` НЕТ включения `includes/header.php` и `includes/footer.php`
- Эти файлы будут подключаться на страницах, а не в системных файлах шаблона
