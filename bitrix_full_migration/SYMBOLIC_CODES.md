# АВТОМАТИЧЕСКАЯ ГЕНЕРАЦИЯ СИМВОЛЬНЫХ КОДОВ

## ✅ УЖЕ НАСТРОЕНО

В файле `/local/php_interface/init.php` уже настроена **автоматическая генерация символьных кодов** для всех элементов инфоблоков.

## 🔧 КАК ЭТО РАБОТАЕТ

### Автоматическая транслитерация

Когда вы создаёте или редактируете элемент инфоблока (дверь, коллекцию, погонаж, фурнитуру), система автоматически:

1. **Проверяет** — если поле "Символьный код" (CODE) пустое
2. **Берёт название** элемента (NAME)
3. **Транслитерирует** его в латиницу
4. **Создаёт SEO-friendly URL** вида: `nazvanie-tovara`

### Примеры генерации

| Название элемента | Символьный код |
|-------------------|----------------|
| Коллекция "Прованс Люкс" | `provans-lyuks` |
| Дверь "Модель Классик" | `model-klassik` |
| Стекло "Матовое белое" | `matovoe-beloe` |

## 📋 ПРАВИЛА ТРАНСЛИТЕРАЦИИ

```php
$arFields["CODE"] = CUtil::translit(
    $arFields["NAME"],
    "ru",
    [
        "max_len" => 100,           // Максимум 100 символов
        "change_case" => "L",       // Нижний регистр
        "replace_space" => "-",     // Пробелы заменяются на дефис
        "replace_other" => "-",     // Спецсимволы заменяются на дефис
        "delete_repeat_replace" => true,  // Убирать повторяющиеся дефисы
        "use_google" => false,
    ]
);
```

## 🌐 ФОРМИРОВАНИЕ URL

### Для дверей

**Шаблон URL:** `/catalog/#COLLECTION_CODE#/#DOOR_CODE#/`

**Пример:**
- Коллекция: "Прованс" → `provans`
- Дверь: "Модель ПГ-01" → `model-pg-01`
- **Итоговый URL:** `/catalog/provans/model-pg-01/`

### Для коллекций

**Шаблон URL:** `/collections/#COLLECTION_CODE#/`

**Пример:**
- Коллекция: "Премиум Классик" → `premium-klassik`
- **Итоговый URL:** `/collections/premium-klassik/`

## ⚙️ НАСТРОЙКА ЧПУ (SEO URLs)

Для корректной работы SEO-friendly URLs настройте **ЧПУ** в Bitrix:

### ШАГ 1: Настроить URL шаблоны инфоблоков

1. Откройте: **Настройки → Настройки продукта → Настройки модулей → Информационные блоки**
2. Включите опцию **"ЧПУ в адресной строке"**

### ШАГ 2: Настроить детальные страницы

#### Для инфоблока "Двери" (ID=2):

**Настройки инфоблока → Вкладка "Настройки":**

- **URL детальной страницы:** `/catalog/#SECTION_CODE_PATH#/#ELEMENT_CODE#/`
- **URL списка:** `/catalog/`

#### Для инфоблока "Коллекции" (ID=1):

- **URL детальной страницы:** `/collections/#ELEMENT_CODE#/`
- **URL списка:** `/collections/`

### ШАГ 3: Создать файл `.htaccess` (если используется Apache)

Файл уже должен быть создан Bitrix, но убедитесь что есть правила:

```apache
RewriteEngine On
RewriteCond %{REQUEST_FILENAME} !-f
RewriteCond %{REQUEST_FILENAME} !-d
RewriteRule ^(.*)$ /bitrix/urlrewrite.php [L]
```

### ШАГ 4: Добавить правила ЧПУ

1. Откройте: **Настройки → SEO → ЧПУ**
2. Добавьте правила:

#### Правило для коллекций:

```
Условие: #^/collections/([a-z0-9-]+)/#
Компонент: dveryaninov:collection.detail
Параметры: ELEMENT_CODE=$1
```

#### Правило для дверей:

```
Условие: #^/catalog/([a-z0-9-]+)/([a-z0-9-]+)/#
Компонент: dveryaninov:door.detail
Параметры: COLLECTION_CODE=$1&DOOR_CODE=$2
```

## 🔄 ОБНОВЛЕНИЕ СУЩЕСТВУЮЩИХ ЭЛЕМЕНТОВ

Если у вас уже есть элементы без символьных кодов:

### Вариант 1: Пакетное обновление через админку

1. Откройте список элементов инфоблока
2. Отметьте элементы без кодов
3. **Действия → Изменить → Изменить**
4. Просто сохраните без изменений — коды сгенерируются автоматически

### Вариант 2: Скрипт массовой генерации

Создайте файл `/local/tools/generate_codes.php`:

```php
<?php
require($_SERVER["DOCUMENT_ROOT"]."/bitrix/modules/main/include/prolog_before.php");

CModule::IncludeModule("iblock");

$IBLOCK_ID = 2; // ID инфоблока (Двери)

$rsElements = CIBlockElement::GetList(
    [],
    [
        "IBLOCK_ID" => $IBLOCK_ID,
        "CODE" => false // Только элементы без кода
    ],
    false,
    false,
    ["ID", "NAME"]
);

$updated = 0;
while ($arElement = $rsElements->Fetch()) {
    $code = CUtil::translit(
        $arElement["NAME"],
        "ru",
        [
            "max_len" => 100,
            "change_case" => "L",
            "replace_space" => "-",
            "replace_other" => "-",
            "delete_repeat_replace" => true,
        ]
    );

    $el = new CIBlockElement;
    if ($el->Update($arElement["ID"], ["CODE" => $code])) {
        $updated++;
        echo "Обновлён элемент ID={$arElement["ID"]}: {$code}<br>";
    }
}

echo "<br>Всего обновлено: {$updated}";
```

**Запуск:** Откройте в браузере `https://ваш-сайт.ru/local/tools/generate_codes.php`

## 🚨 ВАЖНО

1. **Не изменяйте коды вручную** после генерации — это сломает URL
2. **Проверяйте уникальность** — если два элемента имеют одинаковые названия, коды будут дублироваться
3. **Используйте только латиницу и дефисы** в символьных кодах
4. **Очищайте кеш** после изменения кодов: Настройки → Производительность → Очистить кеш

## 📞 УСТРАНЕНИЕ ПРОБЛЕМ

### Проблема: Коды не генерируются автоматически

**Решение:**
1. Проверьте `/local/php_interface/init.php` — есть ли функция `generateSymbolCode()`
2. Убедитесь что обработчики событий зарегистрированы:
   ```php
   AddEventHandler("iblock", "OnBeforeIBlockElementAdd", "generateSymbolCode");
   AddEventHandler("iblock", "OnBeforeIBlockElementUpdate", "generateSymbolCode");
   ```
3. Очистите кеш событий: Настройки → Производительность → Очистить кеш

### Проблема: Кириллица в URL вместо латиницы

**Решение:**
- Убедитесь что в настройках сайта включена транслитерация
- Проверьте что параметр `"change_case" => "L"` установлен в функции `generateSymbolCode()`

### Проблема: 404 ошибка при открытии детальной страницы

**Решение:**
1. Проверьте правила ЧПУ: Настройки → SEO → ЧПУ
2. Убедитесь что компоненты `door.detail` и `collection.detail` настроены корректно
3. Проверьте `.htaccess` — включён ли `mod_rewrite`
