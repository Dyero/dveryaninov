## ⚙️ НАСТРОЙКА КОМПОНЕНТА DOORS.CATALOG

Компонент `dveryaninov:doors.catalog` должен **обязательно выводить товары из инфоблока**.

### Файл: `/local/components/dveryaninov/doors.catalog/component.php`

**Проверьте, что есть запрос к инфоблоку:**

```php
// Получить товары
$arResult["ITEMS"] = [];

$rsDoors = CIBlockElement::GetList(
    ["SORT" => "ASC", "NAME" => "ASC"],
    [
        "IBLOCK_ID" => $arParams["IBLOCK_ID"], // Должно быть 2
        "ACTIVE" => "Y"
    ],
    false,
    ["nPageSize" => $arParams["ELEMENTS_COUNT"]],
    ["ID", "NAME", "CODE", "PREVIEW_PICTURE", "PROPERTY_PRICE", "PROPERTY_COLLECTION"]
);

while ($arDoor = $rsDoors->GetNextElement()) {
    $arFields = $arDoor->GetFields();
    $arProps = $arDoor->GetProperties();

    $arResult["ITEMS"][] = [
        "ID" => $arFields["ID"],
        "NAME" => $arFields["NAME"],
        "CODE" => $arFields["CODE"],
        "PREVIEW_PICTURE" => CFile::GetPath($arFields["PREVIEW_PICTURE"]),
        "PRICE" => $arProps["PRICE"]["VALUE"],
        "DETAIL_PAGE_URL" => "/catalog/код-коллекции/" . $arFields["CODE"] . "/",
    ];
}
```

**Если этого кода нет** — компонент не будет выводить товары!

## 🔧 ДОРАБОТКА ДЛЯ АВТОРИЗАЦИИ

### Вариант 1: Использовать стандартный Bitrix попап

Добавьте в `footer.php` перед `</body>`:

```php
<?php if (!$USER->IsAuthorized()): ?>
<!-- Попап авторизации -->
<div id="auth-popup-container" style="display:none;">
    <?$APPLICATION->IncludeComponent(
        "bitrix:system.auth.form",
        "popup",
        array(
            "REGISTER_URL" => "/auth/registration/",
            "FORGOT_PASSWORD_URL" => "/auth/forgot-password/",
            "PROFILE_URL" => "/personal/",
            "SHOW_ERRORS" => "Y",
        )
    );?>
</div>

<script>
function openAuthPopup() {
    var popup = document.getElementById('auth-popup-container');
    popup.style.display = 'block';
    // Добавьте свою логику показа попапа
}
</script>
<?php endif; ?>
```

### Вариант 2: Интегрировать ваш auth.js с Bitrix

Модифицируйте `/local/templates/dveryaninov/js/auth.js`:

```javascript
// Проверка авторизации
function isBitrixAuthorized() {
    return typeof BX !== 'undefined' && typeof BX.message('USER_ID') !== 'undefined' && BX.message('USER_ID') > 0;
}

// Если пользователь авторизован в Bitrix — обновить UI
if (isBitrixAuthorized()) {
    // Убрать кнопку "Войти", показать кнопку "Профиль"
    // Ваш код
}
```

## 🆘 ЕСЛИ ЧТО-ТО НЕ РАБОТАЕТ

### Проблема: Каталог пустой
**Решение:**
1. Проверьте, что в инфоблоке ID=2 есть товары
2. Проверьте компонент `doors.catalog/component.php` — есть ли запрос к БД
3. Включите режим отладки: Настройки → Режим отладки

### Проблема: Коллекции не показываются на главной
**Решение:**
1. Проверьте, что инфоблок ID=1 существует и содержит коллекции
2. Проверьте свойство PRICE_FROM у коллекций
3. Проверьте PREVIEW_PICTURE загружены

### Проблема: Корзина не работает
**Решение:**
1. Установите модуль Sale: Настройки → Модули
2. Настройте каталог как торговый каталог
3. Проверьте права доступа к модулю

### Проблема: Стили "поехали"
**Решение:**
1. Проверьте, что CSS файлы загружены в `/local/templates/dveryaninov/css/`
2. Очистите кеш браузера (Ctrl+F5)
3. Проверьте консоль браузера на ошибки загрузки файлов
