## Пошаговая инструкция интеграции

### Шаг 1: Загрузка шаблона

1. Подключитесь к серверу по FTP или SSH
2. Загрузите все файлы из `bitrix_full_migration/` в соответствующие директории:

```bash
# Структура шаблона
/local/templates/dveryaninov/header.php
/local/templates/dveryaninov/footer.php
/local/templates/dveryaninov/css/       (вся папка)
/local/templates/dveryaninov/js/        (вся папка)
/local/templates/dveryaninov/images/    (вся папка)
/local/templates/dveryaninov/includes/  (вся папка)
```

3. Проверьте права доступа к файлам:
```bash
chmod 644 /local/templates/dveryaninov/*.php
chmod -R 755 /local/templates/dveryaninov/css/
chmod -R 755 /local/templates/dveryaninov/js/
chmod -R 755 /local/templates/dveryaninov/images/
```

### Шаг 2: Активация шаблона в Bitrix

1. Войдите в админ-панель Bitrix
2. Перейдите: **Настройки → Настройки продукта → Настройки модулей → Главный модуль**
3. Вкладка **"Сайты"** → выберите ваш сайт
4. В поле **"Шаблон по умолчанию"** выберите `dveryaninov`
5. Нажмите **"Сохранить"**

### Шаг 3: Загрузка главной страницы

1. Загрузите файл `index.php` из `bitrix_full_migration/` в корень сайта:
```
/home/bitrix/www/index.php
```

2. Откройте главную страницу сайта в браузере
3. Проверьте:
   - ✅ Отображается хедер с навигацией
   - ✅ Слайдеры работают
   - ✅ Кнопки правильно стилизованы
   - ✅ Отображается футер
   - ✅ Картинки загружаются

**Если кнопки отображаются некорректно:**
- Проверьте, что файл `/local/templates/dveryaninov/css/main.css` загружен
- Откройте консоль браузера (F12) и проверьте, нет ли ошибок загрузки CSS
- Очистите кеш Bitrix: **Настройки → Производительность → Управление кешем → Очистить весь кеш**

### Шаг 4: Создание и загрузка страницы каталога

1. Создайте директорию для каталога:
```bash
mkdir -p /home/bitrix/www/catalog/
```

2. Загрузите файл `catalog.php` как `index.php` в папку `/catalog/`:
```
/home/bitrix/www/catalog/index.php
```

3. Откройте страницу `/catalog/` в браузере
4. Проверьте:
   - ✅ Табы категорий (Все, Межкомнатные двери, Фурнитура и т.д.)
   - ✅ Боковая панель с фильтрами коллекций
   - ✅ Фильтрация работает при клике на табы и фильтры
   - ✅ Карточки товаров отображаются

**Важно для каталога:**
- Файл `shop.js` должен быть загружен - он содержит всю логику фильтрации
- В catalog.php используется инфоблок с ID = 2 (нужно настроить)

### Шаг 5: Создание инфоблоков в Bitrix

Для работы каталога нужно создать инфоблоки:

#### Инфоблок "Коллекции" (ID: 1)
1. **Контент → Типы инфоблоков → Добавить тип**
   - Код: `catalog`
   - Название: `Каталог`

2. **Контент → Структура инфоблоков → Добавить инфоблок**
   - Тип: `Каталог`
   - Название: `Коллекции`
   - Код: `collections`
   - Свойства:
     - `PRICE_FROM` (Число) - Цена от

#### Инфоблок "Каталог товаров" (ID: 2)
1. **Контент → Структура инфоблоков → Добавить инфоблок**
   - Тип: `Каталог`
   - Название: `Каталог товаров`
   - Код: `products`
   - Свойства:
     - `PRICE` (Число) - Цена
     - `COLLECTION` (Строка) - Коллекция
     - `VARIANT` (Строка) - Вариант (ПГ/ПО)
     - `CATEGORY` (Список) - Категория
       - Значения: `doors`, `hardware`, `partitions`, `invisible`
     - `SIZES` (Строка) - Размеры
     - `COATINGS` (Строка) - Покрытия
     - `HEIGHT` (Строка) - Высота
     - `WIDTH` (Строка) - Ширина
     - `THICKNESS` (Строка) - Толщина

### Шаг 6: Страница карточки товара

1. В Bitrix настройте ЧПУ для товаров:
   - **Настройки → Настройки продукта → Настройки модулей → Главный модуль → Вкладка "ЧПУ"**
   - Добавьте правило:
     - Условие: `#^/catalog/([^/]+)/$#`
     - Путь: `/catalog/detail.php?CODE=$1`

2. Создайте файл `/catalog/detail.php`:
```bash
# Загрузите product_detail.php как detail.php
/home/bitrix/www/catalog/detail.php
```

3. Откройте любую карточку товара
4. Проверьте:
   - ✅ Галерея изображений
   - ✅ Выбор размера
   - ✅ Выбор покрытия
   - ✅ Кнопка "Добавить в корзину"
   - ✅ Блок "Другие модели в коллекции"
   - ✅ Модалка "Заказать замер"

### Шаг 7: Очистка кеша и проверка

1. Очистите кеш Bitrix:
   - **Настройки → Производительность → Управление кешем → Очистить весь кеш**

2. Проверьте все страницы в разных браузерах

3. Проверьте мобильную версию (файл `responsive.css` должен работать)

## Troubleshooting (Решение проблем)

### Проблема: Кнопки отображаются некорректно

**Решение:**
1. Проверьте загружены ли CSS файлы:
```bash
ls -la /local/templates/dveryaninov/css/
```
2. Откройте консоль браузера (F12) → вкладка Network
3. Убедитесь что `main.css` и `responsive.css` загружаются со статусом 200
4. Очистите кеш Bitrix

### Проблема: Изображения не отображаются

**Решение:**
1. Проверьте что папка `images` загружена:
```bash
ls -la /local/templates/dveryaninov/images/
```
2. Проверьте пути в коде - должны использовать `SITE_TEMPLATE_PATH`
3. Проверьте права доступа к папке images:
```bash
chmod -R 755 /local/templates/dveryaninov/images/
```

### Проблема: Фильтры в каталоге не работают

**Решение:**
1. Убедитесь что файл `shop.js` загружен и подключен
2. Откройте консоль браузера и проверьте ошибки JavaScript
3. Проверьте что у карточек товаров есть атрибуты `data-collection` и `data-category`

### Проблема: Ошибка "Fatal error: Allowed memory size"

**Решение:**
Эта ошибка уже исправлена в новых файлах `header.php` и `footer.php`. Убедитесь что вы загрузили обновленные версии файлов.

## Важные замечания

1. **ID инфоблоков:** В файлах используются ID = 1 для коллекций и ID = 2 для товаров. После создания инфоблоков проверьте их ID и обновите в файлах если нужно.

2. **Пути к изображениям:** Все пути должны использовать `SITE_TEMPLATE_PATH` константу Bitrix.

3. **Кеш:** После любых изменений очищайте кеш Bitrix.

4. **Символьные коды:** Для всех элементов инфоблоков нужно заполнить символьный код (CODE) для работы ЧПУ.

5. **JavaScript:** Все JS файлы подключаются через `Asset::getInstance()->addJs()` в header.php или напрямую в конце страницы.
